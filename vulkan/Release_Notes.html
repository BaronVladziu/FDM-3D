Vulkan SDK for Linux Release Notes

Vulkan SDK 1.1.82.1 Release Notes

Version 1.1.82.1 for Linux
August 20, 2018

This SDK supports Vulkan API revision 1.1.82. The prior SDK supported Vulkan API revision 1.1.77.

Please refer to the Getting Started Guide for a list of supported Linux distributions.

Drivers: LunarG recommends that you update your Vulkan-capable hardware drivers to the latest recommended version from your hardware vendor. A list of Known Issues appears at the end of this document with more information.

Overview of changes in SDK 1.1.82.1

  • Fixed a bug where SPIR-V validation could incorrectly report valid SPIR-V as invalid. See the github issue for details.

Overview of new features in SDK 1.1.82

  • Bug fixes (see the SDK Components section below)
  • Improved validation coverage
  • The standard Khronos validation layers have been migrated to use the canonical Vulkan VUID values, which are strings in the form of "VUID-vkCmdBlitImage-filter-00236". In doing so, the old integer-based enumerated values have been phased out and are no longer returned by log_msg() calls.
    • Note: Developers keying on these returned values for their development should use the map in the vk_validation_error_messages.h file to obtain the matching Vulkan VUIDs when performing their updates. This file can be found in the the SDK in the VulkanSDK/<SDK version>/Source/layers/ directory and in the layers directory of the Vulkan-ValidationLayers GitHub repository.

Developer Notes

  • The script source/shaderc/update_shaderc_sources.py fails on Arch Linux because python3 is the default python interpreter. To work around this issue, change the line at the top of the file to be python2.

Known Issues

  • It is possible applications will produce a warning to inform the user the VK_LAYER_RENDERDOC_Capture layer is using the deprecated vkGetDeviceProcAdd tag. This tag was deprecated starting with JSON file version 1.1.0. This issue has been reported to the RenderDoc team.
